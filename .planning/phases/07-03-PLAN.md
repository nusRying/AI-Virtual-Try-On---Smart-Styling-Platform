# Plan: 07-03 Storage Layer Abstraction & Cleanup Logic

**Goal**: Abstract image storage to support both local development and future cloud-based scaling (e.g., S3).

## Tasks
1. [x] **Define Storage Interface** - Create `backend/core/storage.py` with a `BaseStorage` class and `LocalStorage` implementation.
2. [x] **Refactor Main API** - Update `backend/main.py` to use the storage service for uploading user photos and merchant garments.
3. [x] **Refactor Celery Tasks** - Update `backend/tasks.py` to handle storage keys and result uploads.
4. [x] **Unified Result URLs** - Ensure all image URLs in API responses are generated through the storage service.
5. [x] **Cleanup Task** - Add a periodic Celery task or an API endpoint to purge old temporary files from `temp/`.

## Success Criteria
- [x] User uploads are saved through the storage service.
- [x] VTO results are uploaded to the storage service and accessible via URL.
- [x] Switching between Local and S3 (mocked) requires only a config change.
- [x] No hardcoded `os.path.join` calls for image storage in `main.py` or `tasks.py`.
